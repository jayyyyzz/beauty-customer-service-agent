# 美妆电商客服 Agent——意图识别专项测评方案

## 1. 测评目标

验证 Agent 能否基于“当前问题 + 历史对话”准确输出：

- `intent_level1`：一级意图
- `intent_level2`：二级意图
- `intent_level3`：三级意图
- `intent_confidence`：意图置信度
- `keywords`：核心关键词
- 是否应该澄清、兜底或继续执行

除了整体准确率，还需要重点检查：

- 高频和低频意图的识别效果是否均衡
- 语义相近的意图能否正确区分
- Agent 能否理解多轮对话中的指代和上下文
- 用户信息不足时能否主动澄清
- 模型输出的置信度是否可信

---

## 2. 测试集设计

针对当前约 22 类一级意图，建议准备 600～1000 条人工标注样本。

### 2.1 基础样本：60%

每个一级意图至少准备 20～30 条样本，覆盖表达清晰、信息完整的常规问题。

```json
{
  "case_id": "intent_0001",
  "history": [],
  "question": "油皮可以用这款面霜吗？",
  "gold_intent_level1": "skin_type",
  "gold_intent_level2": "skin_type.fit",
  "gold_intent_level3": "skin_type.fit.oily",
  "should_clarify": false,
  "difficulty": "easy"
}
```

### 2.2 口语及噪声样本：15%

覆盖错别字、缩写、情绪表达、催促和碎片化消息，例如：

- “这个油皮用会不会闷啊”
- “人呢，我的货到底啥时候发”
- “用了脸红红的，还能退不”
- “？？怎么没物流”

### 2.3 多轮上下文样本：10%

当前问题本身信息不足，必须结合历史对话才能判断意图。

```json
{
  "case_id": "intent_0301",
  "history": [
    {
      "role": "buyer",
      "content": "我昨天买了一瓶修护精华"
    },
    {
      "role": "seller",
      "content": "已经为您发出了"
    }
  ],
  "question": "它现在到哪了？",
  "gold_intent_level1": "logistics",
  "gold_intent_level2": "logistics.tracking",
  "should_clarify": false,
  "difficulty": "hard"
}
```

### 2.4 易混淆样本：10%

重点构造以下意图边界：

- `logistics` 与 `urge_shipment`
- `logistics_delay` 与 `after_sale`
- `safety_allergy` 与 `quality_issue`
- `efficacy` 与 `skin_concern`
- `usage` 与 `routine`
- `price` 与 `gift_sample`
- `after_sale` 与 `quality_issue`

### 2.5 澄清与兜底样本：5%

覆盖信息不足、意图模糊、多意图冲突和领域外问题，例如：

- “这个怎么样？”
- “帮我处理一下。”
- “还是之前那个问题。”
- 与美妆电商完全无关的问题
- 同时包含多个无法确定主次的诉求

这类样本的正确结果可能是触发澄清，而不是强制选择某个业务意图。

```json
{
  "case_id": "intent_0501",
  "question": "帮我处理一下",
  "should_clarify": true,
  "acceptable_intents": [
    "other",
    "after_sale"
  ]
}
```

## 3. 核心指标

### 3.1 一级、二级、三级 Accuracy

分别计算每个意图层级判断完全正确的比例：

```text
Accuracy = 判断正确的样本数 / 总样本数
```

必须分别输出：

- Level-1 Accuracy
- Level-2 Accuracy
- Level-3 Accuracy
- 三级路径完全准确率

三级路径完全准确要求一级、二级和三级意图全部正确。例如：

```text
skin_type
→ skin_type.fit
→ skin_type.fit.oily
```

任何一个层级错误，完整路径都记为错误。

### 3.2 Precision、Recall 和 Macro-F1

```text
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 = 2 × Precision × Recall / (Precision + Recall)
Macro-F1 = 所有意图 F1 的平均值
```

Macro-F1 不受类别样本量影响，能够避免物流、优惠等高频意图掩盖发票、赠品等低频意图的错误，是本项目的核心指标之一。

### 3.3 Micro-F1

汇总所有类别的 TP、FP 和 FN 后计算，反映模型整体识别表现。

单标签多分类场景下，Micro-F1 通常接近 Accuracy，但建议保留，以支持后续多意图识别扩展。

### 3.4 Weighted-F1

根据各类别的实际样本数对 F1 加权，反映真实业务流量分布下的整体效果。

最终报告至少输出以下表格：

| 层级 | Accuracy | Macro-F1 | Micro-F1 | Weighted-F1 |
|---|---:|---:|---:|---:|
| Level-1 |  |  |  |  |
| Level-2 |  |  |  |  |
| Level-3 |  |  |  |  |

## 4. 分层意图路径得分

三级意图完全匹配比较严格，可增加层级路径得分作为辅助分析指标：

```text
一级正确：0.4 分
二级正确：再得 0.3 分
三级正确：再得 0.3 分
```

| 预测结果 | 路径得分 |
|---|---:|
| 一级、二级、三级全部正确 | 1.0 |
| 一级、二级正确，三级错误 | 0.7 |
| 只有一级正确 | 0.4 |
| 一级错误 | 0 |

该指标仅用于分析错误严重程度，不能替代 Level-3 Accuracy。

## 5. 澄清策略评测

当前 Agent 使用置信度阈值判断是否触发澄清，因此需要单独评测澄清策略。

### 5.1 评测指标

- **澄清 Precision**：触发澄清的样本中，真正需要澄清的比例
- **澄清 Recall**：需要澄清的样本中，成功触发澄清的比例
- **错误放行率**：本应澄清，却直接执行错误任务的比例
- **过度澄清率**：用户问题已经明确，却仍要求用户补充信息的比例

错误放行可能导致查询错误订单或执行错误业务，风险高于普通分类错误。

### 5.2 建议目标

| 指标 | 建议目标 |
|---|---:|
| 澄清 Precision | ≥ 85% |
| 澄清 Recall | ≥ 85% |
| 错误放行率 | ≤ 5% |
| 过度澄清率 | ≤ 10% |

### 5.3 阈值选择

不要直接假定当前 `0.65` 是最优阈值，应测试以下候选值：

```text
0.50、0.55、0.60、0.65、0.70、0.75、0.80
```

比较不同阈值下的澄清 Precision、Recall、错误放行率和过度澄清率，选择符合业务风险要求的阈值。

## 6. 置信度校准

模型返回的 `intent_confidence` 属于模型自我判断，并不天然可信，需要验证置信度与真实正确率是否匹配。

### 6.1 ECE

Expected Calibration Error 用于判断模型置信度是否可靠。

将预测结果按置信度分桶：

```text
0.5～0.6
0.6～0.7
0.7～0.8
0.8～0.9
0.9～1.0
```

对比每个区间内的平均置信度和真实准确率。

### 6.2 Brier Score

```text
Brier Score = 平均值((预测置信度 - 实际结果)²)
```

Brier Score 越接近 0，说明置信度越可靠。

建议目标：

- ECE ≤ 0.10
- Brier Score ≤ 0.15
- 单独统计置信度 ≥ 0.9 但预测错误的案例

## 7. 错误分析

### 7.1 混淆矩阵

统计真实意图与预测意图之间的混淆关系：

| 真实意图 | 常见错误预测 | 错误数量 |
|---|---|---:|
| urge_shipment | logistics |  |
| safety_allergy | quality_issue |  |
| usage | routine |  |
| after_sale | logistics_delay |  |

### 7.2 分场景准确率

至少按照以下维度对结果进行切片：

- 无历史对话与有历史对话
- 正常表达与口语噪声
- 单意图与多意图
- 正常情绪与负面情绪
- 信息完整与需要澄清
- 高频意图与低频意图
- Easy、Medium、Hard

通过切片分析判断问题来自意图体系、Prompt、上下文处理还是置信度策略。

### 7.3 错误类型

建议为每个错误样本添加以下标签之一：

- `taxonomy_boundary`：意图体系边界不清
- `context_missing`：未正确使用历史对话
- `multi_intent`：多意图处理失败
- `noise_robustness`：错别字、口语或情绪导致误判
- `over_confident`：错误但置信度过高
- `under_confident`：正确但置信度过低
- `clarify_missed`：应该澄清但直接执行
- `over_clarify`：无需澄清却触发澄清
- `output_format`：输出字段或 JSON 格式错误

## 10. 评测执行流程

1. 冻结当前意图分类标准和 Prompt 版本。
2. 人工构建并复核测试集，禁止使用测试数据优化 Prompt 后不重新划分数据。
3. 使用固定模型、固定温度和固定参数运行全部测试样本。
4. 保存模型原始输出、解析结果、耗时和异常信息。
5. 计算各层级 Accuracy、F1、澄清指标和置信度指标。
6. 生成混淆矩阵以及各业务场景的切片报告。
7. 对高频错误进行归因，修改意图体系、Prompt或路由策略。
8. 使用独立回归测试集验证优化是否真正有效。

建议保存以下版本信息：

- 测试集版本
- 意图体系版本
- Prompt 版本
- 模型名称及版本
- 模型参数
- 代码提交版本
- 评测时间

## 11. 最终评测报告结构

```text
1. 测试集规模与意图分布
2. Level-1/2/3 Accuracy
3. Macro-F1、Micro-F1、Weighted-F1
4. 各意图 Precision、Recall、F1
5. 混淆矩阵
6. 多轮、噪声、易混淆样本表现
7. 澄清策略表现
8. 置信度校准结果
9. 高置信度错误案例
10. Prompt、意图体系及路由优化建议
```
