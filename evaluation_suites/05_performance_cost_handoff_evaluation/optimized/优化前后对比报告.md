## 本轮追加优化的直接收益

上一轮优化后仍存在 Knowledge P95 14.63s、Hybrid P95 12.02s、ES 检索 P95 2.46s 的问题。本轮将 Elasticsearch 查询改为进程内复用 HTTP 连接池后：

- ES 检索 P95 从 2.461s 降至 0.438s，下降 82.19%。
- Knowledge P95 从 14.633s 降至 6.632s，下降 54.68%，由未通过转为通过。
- Hybrid P95 从 12.024s 降至 5.736s，下降 52.29%，由未通过转为通过。
- 4 并发吞吐从 0.675 RPS 提升至 1.531 RPS，再提升 126.76%。
- 总成本从 $0.00075935 降至 $0.00063736，再下降 16.06%。

## 索引发布变化

| 项目 | 优化前入库行为 | 最终优化后行为 |
|---|---|---|
| HTTP 请求 | 每批创建网络请求，受系统代理探测影响 | 复用连接池，关闭本机代理探测 |
| 变更判断 | 内容哈希变化即重算向量 | 区分向量输入变化与普通元数据变化 |
| FAQ 同步 | 17,453 条被误判为需重新编码，15 分钟未完成 | 39 秒完成元数据同步，无向量重算 |
| 二次校验 | 未完成 | 25,520 条全部一致，0 条变更 |

## 代价与权衡

- 默认关闭 LLM Rerank 显著降低延迟与 Token，但仍需持续运行完整检索质量基准，防止复杂问题召回退化。
- 显式意图快速路径只覆盖边界清晰表达；歧义输入继续交给 LLM，避免规则误分类。
- 连接池成为长期资源，需要在 Web 服务关闭时补充统一释放机制，并在长稳压测中监控连接数。
- 当前人工转接结果来自 24 条专项样本，不能替代真实人工运营数据。
