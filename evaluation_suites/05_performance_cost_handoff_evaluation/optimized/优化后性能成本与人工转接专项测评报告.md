# 优化后性能、成本与人工转接专项测评报告

测评日期：2026-07-16  
结论：**本专项设定的请求成功率、Knowledge、Hybrid、Business API、Clarify、人工转接和故障降级目标全部通过。** 当前主要剩余项转为 TTFT、长时间稳定性和更大规模质量验证。

## 1. 本轮最终实施优化

- 新增确定性人工转接策略，覆盖用户主动要求人工、账户安全、监管投诉、高金额争议、重复失败和护肤不良反应。
- 人工工单保存历史消息、订单号、策略信号、失败阶段和人工下一步建议；LLM、ES 或业务工具失败均能安全转人工。
- 纯业务路由跳过 RAG；明确的流程、肤质、物流、质量、发票、保质期和领域外问题使用规则快速路径。
- 默认关闭 LLM Rerank，保留条件开关和词法安全回退；ES 候选参数收缩至 12/120。
- 护肤流程和保质期等强关键词问题采用 BM25；回答限制为 220 个汉字、最大 500 Token。
- 业务查询结果使用确定性模板渲染，避免模型补写订单、物流和退款事实。
- 模型迁移至 `deepseek-v4-flash`。
- Elasticsearch 查询从逐次创建标准网络客户端改为进程内复用 HTTP 连接池，并关闭本机请求的系统代理探测。
- ES 入库同样复用连接池；新增 `embedding_input_hash`，把“需要重算向量”和“只需更新元数据”分开。
- 完成 17,453 条 FAQ 元数据快速同步，无需重新计算向量；再次执行增量同步为 0 条变更。

## 5. 故障降级

|---|---|
